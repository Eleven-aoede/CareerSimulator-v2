# 职业模拟故事生成

## 任务定义

根据输入的岗位信息和用户画像，生成一份可被前端直接读取和运行的职业模拟交互脚本 JSON。

最终产物必须是：
- 可被前端直接读取的故事脚本 JSON
- 不是自然语言说明
- 不是给人看的 Markdown
- 不是任务清单式 JSON

## 输入

```json
{input_json}
```

## 输入处理规则

### 岗位信息

至少使用这些字段：role_name、job_tasks、industry、company_size。
`additional_context` 可选，但如果存在，应尽量吸收到故事背景中。

### 用户画像

如果 `profile_skipped` 为 `false`，应让用户画像真实影响：
- 更自然出现的选项措辞
- 节点里的心理活动
- 哪些事件更容易让用户紧张
- 哪些事件更容易让用户获得成就感
- 结局里的匹配分析理由

如果 `profile_skipped` 为 `true`，按"中性默认用户"生成：选项保持均衡、心理活动保持普适、匹配分析更弱。

## 输出格式

必须输出以下 JSON 结构（不要包含任何其他内容，直接输出 JSON）：

```json
{
  "meta": {
    "eyebrow": "CAREER SIMULATOR / STORY",
    "title": "职业模拟：{岗位名称}<br />交互脚本",
    "plainTitle": "职业模拟：{岗位名称}",
    "description": "一段简短说明。",
    "settings": [
      { "label": "行业", "value": "..." },
      { "label": "公司规模", "value": "..." },
      { "label": "你的身份", "value": "..." },
      { "label": "故事时段", "value": "..." }

  "story": {
    "intro": { ... },

    "taskAction": { ... },
    "taskEmotion": { ... },
```

## 固定生成架构

### intro（开场导入）
交代：岗位所在行业与公司环境、用户进入的身份位置、第一阶段大致面对的节奏。

### node1-node3（适应事件）
让用户进入真实的第一批工作现场：第一次接到任务、第一次参加例会、第一次看资料、第一次和带教确认方向。

### taskAction/taskEmotion/taskDifficulty（典型任务 mini 模拟）
- taskAction：给出岗位最常见任务的一小段具体现场 + 3条左右原始素材 + "你会先怎么处理"的问题
- taskEmotion：记录用户面对核心任务时的即时情绪
- taskDifficulty：1-5 五点评分，每个选项带白话解释

### node4（压力事件）
出现一个高概率压力源：deadline、返工、模糊需求、客户压力等。

### node5（成就或失误事件）
让用户感受到这份工作的满足感或明显紧绷。

### endings（结局）
结局必须是描述性的，不是裁判式的。要说明：用户在这份工作里大概会怎样生活和感受、哪些部分顺手、哪些部分持续拉扯。

## 节点写法要求

- 一个节点只推进一个很小的瞬间
- 先有现场，再有情绪，再有选择
- 除 intro 外，普通节点的 paragraphs 默认至少 2 段
- 先写眼前发生了什么，再写用户此刻什么感觉，再进入选择

## 选项写法要求

- 每题默认 2-3 个选项
- 每个选项只表达一个动作或一种即时感受
- 同一题里的选项要平行
- 用户第一次接触这个行业也应能看懂
- 像人在工位上会说的话，不是写给领导的周报语言

### effect 数值规则
- 取值范围：-1, 0, 1, 2
- fit：选择和岗位要求的贴合度
- stress：选择是否让压力更往上顶
- growth：选择是否推动学习或适应

## 硬约束

1. 不能写成任务清单（simulation_flow / step_id / scenario+description+options）
2. 每个节点先有场景再有选择
3. 选择必须是小步动作
4. 必须有现场感（办公室、会议室、消息弹窗、走廊、工位）
5. taskAction 必须给出真实可处理的原始素材

## 生成流程

1. 提炼岗位画像（工作节奏、典型产出、协作结构、压力来源、成就来源）
2. 提炼用户体验倾向（自然应对方式、压力触发点、成就感反馈、沟通行动姿态）
3. 做岗位×用户交叉判断（贴合还是拉扯、哪些特质帮助适应、哪些放大压力）
4. 生成连续职业旅程

## 输出要求

直接输出完整的 JSON，不要包含任何解释文本或 markdown 代码围栏。
