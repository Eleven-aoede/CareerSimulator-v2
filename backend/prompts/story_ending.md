## 结局等级

`{ending_bucket}`

- high（fit >= 9）：用户在这份工作中整体贴合，大多数时候能自然应对
- mid（3 <= fit < 9）：部分贴合部分拉扯，有些地方顺手有些地方持续消耗
- low（fit < 3）：明显不适配，核心日常和用户自然状态长期冲突

```json
{task_recap}
```

## 结局写法要求

结局必须是描述性的，不是裁判式的。要说明：
- 用户在这份工作里大概会怎样生活和感受
- 哪些部分顺手
- 哪些部分持续拉扯
- 这些体验和用户之前做出的选择之间的关系

结局段落至少 3 段：
1. 总结这段职业体验的整体感受
2. 具体回顾哪些节点的选择体现了贴合或冲突
3. 给出一个面向未来的开放性思考

## 输出格式

直接输出以下 JSON 结构，不要包含任何其他文本：

```json
{
  "title": "结局标题",
  "paragraphs": ["段落1", "段落2", "段落3"],
  "matchLabel": "匹配程度标签（如：自然贴合 / 有所拉扯 / 明显冲突）",
  "reason": "一句话总结匹配原因"
}
```

## 硬约束

1. 不写"你不适合这份工作"或"你很适合"——要写出具体的日常体验
2. 必须引用用户在模拟中做出的具体选择
3. 语言保持温暖、不评判
4. 直接输出 JSON，不要有 markdown 代码围栏或解释文本
