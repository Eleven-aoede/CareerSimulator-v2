# HTML Flow

```mermaid
flowchart TD
    A[首页<br/>标题 / 小可形象 / 引导语 / 开始游戏 / 直接开始模拟 / 历史记录]
    B[开始游戏]
    B2[直接开始模拟<br/>读取最近一次保存的岗位信息]
    C[开场页<br/>显示小可固定开场白 + 岗位信息收集按钮]
    D[逐题收集岗位信息<br/>岗位名称 / JD / 行业 / 公司规模 / 补充信息]
    E[阶段切换页<br/>显示用户画像收集引导语]
    F{是否开始画像收集}
    G[逐题收集用户画像<br/>一个问题一页 / 点击进入下一题]
    H[生成输入 JSON<br/>包含 user_profile + job_input]
    I[结果页<br/>显示固定承接语 / 下载 JSON / 浏览器存档]
    J[上传故事脚本 JSON<br/>载入交互文本]
    K[开始模拟]
    L[首页历史记录]
    M[查看历史 JSON]
    N[下载历史 JSON]

    A --> B
    A --> B2
    B --> C
    B2 --> I
    C --> D
    D --> E
    E --> F
    F -->|开始画像收集| G
    F -->|跳过画像收集| H
    G --> H
    H --> I
    I --> J
    J --> K
    I --> L
    L --> M
    L --> N
```

## Notes

- 首页只负责进入流程和查看历史记录。
- 首页现在只保留“开始游戏”一个入口。
- 首页现在包含两个入口：开始游戏、直接开始模拟。
- 直接开始模拟会优先读取最近一次保存的岗位信息和画像，再进入结果页继续上传故事脚本。
- 岗位信息收集和画像收集都采用“一题一页”的固定节奏，顺序是先岗位、后画像。
- 岗位信息结束后有一个明确的阶段切换页，用来提示用户接下来进入画像收集。
- 结果页先生成输入 JSON，再提供故事脚本上传入口，上传后直接进入模拟。
- 历史记录中的每一条岗位信息都可以查看、下载或删除。
- 下一步链路应为：
  `输入 JSON -> 小可生成故事脚本 JSON -> 模拟器读取故事脚本 JSON`
